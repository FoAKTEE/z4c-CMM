==== Description of expected options:
Find the solution to an XCTS problem.

Options:
  ResourceInfo:
    type=ResourceInfo
    Options for allocating resources. This information will be used when
    placing Array and Singleton parallel components on the requested
    resources.

  RadiallyCompressedCoordinates:
    type=Auto
    Define radially compressed coordinates for visualizing large outer radii.

  DomainCreator:
    type=DomainCreator
    The domain to create initially

  Parallelization:
    Options related to parallelization aspects of the simulation.

  NonlinearSolver:
    The iterative nonlinear solver

  LinearSolver:
    The iterative Krylov-subspace linear solver

  Amr:
    Options for adaptive mesh refinement (AMR)

  Observers:
    Options for recording data

  Discretization:
    Options for the discretization of the elliptic equations

  InitialGuess:
    type=InitialGuess
    The initial guess for the elliptic solve. The solve converges faster if
    the initial guess is close to the solution and may not converge at all if
    the initial guess is too far away.

  RandomizeInitialGuess:
    type=Auto
    Add uniform random noise to variables.

  Background:
    type=Background
    The variable-independent part of the equations that define the problem to
    solve (along with the boundary conditions).

  EventsAndTriggersAtIterations:
    type=EventsAndTriggers
    Events to run at triggers

  PhaseChangeAndTriggers:
    type=[TriggerAndPhaseChanges, ...]
    A collection of pairs of triggers and collections of phase change objects
    to determine runtime phase control-flow decisions. The order of the phase
    change objects determines the order of the requests processed by the Main
    chare during phase change arbitration.

  BuildMatrix:
    Options for building the explicit matrix representation of the linear
    operator. This is done by applying the linear operator to unit vectors
    and is useful for debugging and analysis only, not to actually solve the
    elliptic problem (that should happen iteratively).
